安全提示

本文涉及网络配置相关内容，请合理使用，遵守相关法律法规。

在这个互联网互联互通的时代,我们经常需要处理各种不同格式的网络配置。sub2clash 作为一款强大的订阅转换工具,能够优雅地将各类订阅链接(支持 Shadowsocks、ShadowsocksR、Vmess、Vless、Trojan、Hysteria 等多种协议)转换为 Clash 配置文件。无论是在软路由上统一管理多个订阅，还是需要将不同协议的节点整合到一起，亦或是想要使用 Clash 强大的分流功能，sub2clash 都能完美满足这些需求。本文将带您探索如何在 Linux 系统上部署这个实用工具,让您的网络配置管理变得轻松自如。

系统要求

  • 支持的架构：amd64, arm64
  • 操作系统：Linux (包括 OpenWrt)
  • 内存要求：≥ 64MB
  • 存储空间：≥ 10MB

前期准备

在开始部署之前,请确保系统已安装以下软件:

  • screen (用于保持服务运行)
  • curl (用于下载文件)
  • cron (用于定时任务)
  • bash (推荐安装,提供更好的脚本兼容性)

使用说明

基本操作

  • 手动更新订阅: /root/update-clash.sh
  • 查看转换服务: screen -r sub2clash
  • 退出 screen 会话: 按 Ctrl+A 然后按 D
  • 访问控制面板: http://<your-server-ip>:9090/ui/ (请将 替换为你的服务器 IP 地址)

配置文件

  • 配置文件路径: /root/.config/clash/config.yaml
  • 重要参数说明：
    • port: HTTP 代理端口
    • socks-port: SOCKS5 代理端口
    • mixed-port: 混合代理端口
    • allow-lan: 是否允许局域网访问
    • mode: 代理模式(Rule/Global/Direct)

安全提示

  1. 请勿在公网环境直接暴露 sub2clash 服务端口
  2. 建议配置访问控制，限制可访问的 IP 范围
  3. 定期更新 sub2clash 版本以修复潜在安全问题
  4. 请妥善保管您的订阅链接

注意事项

  1. 请确保系统已正确安装 screen
  2. 确保 sub2clash 服务正常运行
  3. 定时更新时间可根据需要调整
  4. 首次部署后需手动执行一次更新脚本

常见问题

  1. 服务无法启动

    • 检查端口是否被占用：lsof -i:8011
    • 确认执行权限是否正确：chmod +x sub2clash
    • 查看日志：tail -f clash.log

  2. 订阅更新失败

    • 检查网络连接：ping github.com
    • 验证订阅链接有效性
    • 确认 sub2clash 服务状态：ps aux | grep sub2clash
    • 检查存储空间：df -h

维护建议

  1. 定期检查日志文件大小，避免占用过多存储空间
  2. 建议保留配置文件备份
  3. 监控服务运行状态，可配置监控告警
  4. 适时更新 sub2clash 版本以获得新特性和修复
